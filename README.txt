Rita de Cássia Dias Rodrigues Monteiro 

Esta atividade prática de Git e GitHub envolve as seguintes etapas:

Criar um repositório público no GitHub com o nome atividade-git-nome-sobrenome.
Clonar o repositório para sua máquina local.
Criar um arquivo README.txt com seu nome, descrição da atividade e comandos Git usados (com explicações).
Criar a pasta projeto, com um index.html e um .gitignore (para ignorar arquivos .log).
Realizar operações Git básicas (git add, git commit, git push, git branch, git checkout, git merge), registrando cada passo com um printscreen.
Editar o index.html na branch desenvolvimento, fazer um commit e mesclar com a branch main.
A entrega inclui o link do repositório no GitHub e a documentação dos passos e comandos no README.txt.

lista de comandos:

git add: Adiciona arquivos ao staging area (preparação para o commit).
git commit: Registra as alterações no histórico do repositório local.
git push: Envia as alterações do repositório local para o repositório remoto (GitHub).
git branch: Cria uma nova ramificação (branch).
git checkout: Muda para uma branch específica.
git merge: Combina as alterações de uma branch em outra.
